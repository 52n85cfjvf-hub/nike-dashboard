| Квартал | Месяцы | Пример |

## 2. Извлечение данных

### Источники
PDF-файлы квартальных пресс-релизов NIKE содержат страницы как растровые изображения.
Текстовый слой отсутствует (PyMuPDF возвращает пустой текст).

### Процесс
1. Рендеринг каждой страницы PDF в PNG (PyMuPDF, масштаб 1.5×)
2. Визуальная инспекция страниц для нахождения таблицы "DIVISIONAL REVENUES"
3. Извлечение значений из колонки "THREE MONTHS ENDED" (не накопленные данные)
4. Верификация по строке "Total" каждого региона

### Верификация
Каждое значение верифицировано перекрёстной проверкой:
- Сумма Footwear + Apparel + Equipment = Total региона
- Сумма всех регионов ≈ Total NIKE Brand (за вычетом Global Brand Divisions)

## 3. Расчёт показателей

### Общая анализируемая выручка
```
total_analyzed = сумма выручки 12 направлений (3 × 4) за квартал
```
Не включает: Converse, Corporate, Global Brand Divisions.

### Доля направления
```
revenue_share_pct = revenue_usd_mn / total_analyzed_revenue_usd_mn × 100
```

### Рост год к году
```
revenue_yoy_growth_pct = (revenue[t] / revenue[t-4] − 1) × 100
```
Сравнение с тем же финансовым кварталом предыдущего года.
Для Q1–Q4 FY2023 значение равно NaN (нет данных за год назад).

### Агрегация при мультивыборе
**ВАЖНО:** При выборе нескольких продуктов/регионов:
1. Сначала суммируется выручка всех выбранных направлений
2. Затем рассчитываются доля и рост из агрегированных значений

Нельзя суммировать доли или темпы роста напрямую.

## 4. Расчёт сезонных индексов

### Алгоритм
Для каждого из 12 сочетаний (product × region):

```python
avg_q[i] = mean(revenue за все годы, где fiscal_quarter == i)  # i = 1,2,3,4
overall_avg = mean(avg_q[1], avg_q[2], avg_q[3], avg_q[4])
seasonal_index[i] = avg_q[i] / overall_avg
```

### Интерпретация
- `seasonal_index > 1`: квартал исторически выше среднего
- `seasonal_index < 1`: квартал исторически ниже среднего
- Среднее всех индексов для одного сочетания ≈ 1.0

### Применение в дашборде
- Определение лучшего квартала для каждого направления
- Формирование текстовых выводов (автоматически из данных)
- Проверка разумности прогнозов

## 5. Методология прогноза

### Архитектура
Прогнозируется **каждое из 12 направлений отдельно** (не суммарная выручка).
Это обеспечивает:
- корректную квартальную сезонность для каждого направления
- возможность агрегировать прогнозы при любых фильтрах
- корректный расчёт прогнозной доли

### Основная модель: Holt-Winters
```python
ExponentialSmoothing(
    series,
    trend="add",          # аддитивный тренд
    seasonal="add",       # аддитивная сезонность
    seasonal_periods=4,   # 4 финансовых квартала
    initialization_method="estimated"
).fit(optimized=True)
```
Сезонная компонента встроена в модель — повторно добавлять сезонный
индекс поверх прогноза нельзя (двойной учёт).

### Резервный метод: Сезонный наивный
```python
forecast[Q(t)] = actual[Q(t−4)]
```
Применяется при: < 8 наблюдений, неположительных значениях,
или если Holt-Winters даёт MAE выше сезонного наивного.

### Выбор модели
Простая временна́я проверка (time-series cross-validation):
1. Исключить последние 4 квартала из обучения
2. Обучить обе модели на ранних данных
3. Сравнить MAE на исключённых кварталах
4. Выбрать модель с меньшим MAE

### Агрегация прогнозов
```
forecast_selected = sum(forecast[d] for d in selected_directions)
forecast_total    = sum(forecast[d] for d in all_12_directions)
forecast_share    = forecast_selected / forecast_total × 100
```

### Горизонт
4 квартала после последнего фактического квартала.

## 6. Ограничения

| Ограничение | Описание |
|-------------|----------|
| История | Только 13 кварталов — мало для обучения сложных моделей |
| Сезонность | 3 наблюдения на квартал (нижняя граница) |
| Валюта | Выручка в USD по текущим курсам (не нейтрализована) |
| Scope | Только выручка, не прибыль или маржинальность |
| Converse | Исключён — не входит в NIKE Brand |
| Прогноз | Учебный характер, не инвестиционная рекомендация |
